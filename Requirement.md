## 프로젝트 개요

- 서비스명 : invest72
- 핵심 가치 : 파편화된 금융 상품 데이터를 통합하여 사용자에게 미래 자산 성장 경로를 시각적으로 제시하고, 저축 및 투자 동기를 부여함.
- 타겟 사용자 : 사회 초년생 및 금융 입문자

## 주요 기능 상세

### 인증 및 보안

- 소셜 로그인 : 구글, 네이버 소셜 플랫폼 연동

### 금융 투자 상품 관리 (CRUD)

- 현금(Cash) : 유동 자금 관리(이율 0% 적립식).
- 예금/적금 : 단리 및 복리 계산 로직 적용
- 유연한 확장성 : 주식, 채권 등 추후 상품군 추가가 용이한 데이터 구조.

금융 투자 상품
- 예금
    - 투자 유형: 예금
    - 금액 유형: 일시불
    - 금액(원금)
    - 기간
        - 기간 데이터는 개월수(month)로 저장
        - 사용자는 년 데이터를 입력할 수 있어도 저장할때는 개월수로 저장
    - 이자 유형
        - 단리, 복리
        - 복리는 월복리 기준으로 작동
    - 세금 타입
        - 일반과세, 비과세, 세금 우대
        - 일반과세는 15.4% 고정, 비과세는 0% 고정
    - 세율
        - 세율은 배율 단위로 저장한다. 예를 들어 15.4%는 0.154 값으로 저장
	- 시작일
- 적금
    - 투자 유형: 적금
    - 금액 유형: 월 적립액
    - 금액(원금)
        - 매달 납부하는 월 적립금액에 해당됨
    - 기간
        - 기간 데이터는 개월수(month)로 저장
        - 사용자는 년 데이터를 입력할 수 있어도 저장할때는 개월수로 저장
    - 이자 유형
        - 단리, 복리
        - 복리는 월복리로 작동
    - 세금 타입
        - 일반과세, 비과세, 세금우대
    - 세율
        - 세율은 배율 단위로 저장한다. 예를 들어 15.4%는 0.154 값으로 저장
    - 시작일
- 현금
    - 투자 유형: 현금
    - 금액 유형: 일시불
    - 금액(원금)

금융 투자 상품 데이터 제약조건
- 이름
	- 길이 범위 : 1 ~ 100
- 상품 유형
	- 값 범위 : CASH, DEPOSIT, SAVINGS
	- 길이 범위 : 최대 100
- 금액
	- 금액의 길이는 정수부분 17자리까지, 소수부분 2자리까지 제약됩니다.
		- 예를 들어 원화(KRW)기준 99999조까지 저장 가능합니다.
		- 금액의 값이 123.456라면 데이터베이스에는 123.45와 같이 저장됩니다.
	- 값 범위
		- 원화(KRW) : 0 ~ 10000조
		- 달러(USD) : 0 ~ 10000조 달러
	- 데이터베이스에는 환율을 고려하지 않고 정수부분 최대 19자리, 소수부분 최대 2자리가찌 저장합니다.
		- 예를 들어 10000조달러를 원화로 변경하여 저장시 원달러 환율이 1500원이라면 15,000,000조원이 저장되지 않을 수 있습니다.
- 기간
	- 값 범위 : 0 ~ 9999
- 연이율
	- 값 범위 : 0.0 ~ 9.9999 (999.99%)
- 이자 유형
	- 값 범위 : NONE, SIMPLE, COMPOUND
	- 길이 범위 : 최대 100
- 세금 유형
	- 값 범위 : STANDARD, NON_TAX, TAX_BENEFIT, NONE
	- 길이 범위 : 최대 100
- 세금 이율
	- 값 범위 : 0.0 ~ 9.9999

금융 투자 상품 API
- 상품 생성(Create) : `POST /api/v1/products`
- 상품 조회(Search)
    - 목록 조회 : `GET /api/v1/products`
    - 상세 조회 : `GET /api/v1/products/{id}`
    - 상품 수익 조회 : `GET /api/v1/products/{id}/calculate`
- 상품 수정(Updat) : `PUT /api/v1/products/{id}`
- 상품 삭제(Delete) : `DELETE /api/v1/products/{id}`

상품 수정 제약 조건
- 사용자가 상품 수정시 상품의 유형(현금/예금/적금)을 변경할 수 없습니다.
	- 예를 들어 현금 상품을 적금 상품으로 변경할 수 없습니다.


### 금융 상품 목록 페이지
- 사용자는 자신이 등록한 금융 상품 목록들을 확인할 수 있다.
- 금융 상품 목록의 컬럼은 상품명, 금리, 시작일자, 만기일, 현재 잔액(원금), 예상이자, 진행률(Progress Bar, D-120)
	- 예를 들어 예금 상품의 경우 "농협 정기예금", "5%", "2026-12-31", "1,000,000원", "50,000원", Progress Bar(D-120)
- 적금 상품의 경우에는 월 납입액을 추가로 표시해야 한다.
- 현금 상품의 경우에는 금리는 0%, 만기일 없음, 현재 잔액(원금) 표시, 예상이자 0%, 진행률 없음
- 금융 상품 목록 정렬
	- 금융 상품 목록의 정렬 기준은 가입일 최신순입니다. (시작일자을 기준으로 내림차순)
	- 가입일이 동일하다면 만기일이 임박한 순서로 정렬합니다. (만기일을 기준으로 오름차순)
	- 만기일도 동일하다면 투자금액이 큰 순서로 정렬합니다. (금액을 기준으로 내림차순)
	- 최종적으로 시스템 등록 순서에 따라 고정 (생성시간을 기준으로 오름차순)

### 자산 성장 시뮬레이션 및 시각화

- 대시보드 : 사용자는 홈 화면에서 사용자 등록한 금융 상품들의 합계 대시보드를 확인할 수 있다.
- 통합 자산 그래프 : 등록된 모든 상품의 미래 가치를 합산한 통합 성장 곡선 출력(Recharts 활용)
- 상세 리포트 : 연도별/월별 원금 및 이자 수익 상세 데이터 제공

### 72의 법칙 위젯

- **개요** : 복리 이율을 바탕으로 자산이 2배가 되는 기간을 즉시 계산해주는 대시보드 도구.
- **주요 기능**
    - **자동 계산** : 사용자가 등록한 상품의 평균 수익율 또는 사용자가 입력한 특정 이율을 바탕으로 자산이 2배가 되는데 걸리는 “년수” 산출(72 / 이율 = 기간).
    - **인사이트 제공** : "현재 5% 수익률로는 자산이 2배가 되는 데 14.4년이 걸립니다. 수익률을 2% 높이면 기간을 4년 단축할 수 있습니다!"와 같은 가이드 메시지 출력.
    - **시뮬레이션 슬라이더** : 이율을 슬라이더로 조절하며 기간의 변화를 실시간으로 확인하는 인터랙티브 UI.

## 기술 스택
- Frontend : React
- Backend : Spring Boot, PostgreSQL
- Deployment : Firebase / Cloud Run
- Visual : Recharts / Chart.js

## 기대 효과

- 동기 부여 : 막연한 저축이 아닌, 시각화된 미래 자산을 보며 저축 의지를 고취함.
- 의사 결정 보조 : 상품별 이자 방식(단리/복리)에 따른 최종 수익 차이를 직관적으로 비교하여 더 나은 금융 선택을 유도함.
